# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository purpose

Personal macOS dotfiles for zsh, Ghostty terminal, and Homebrew. There is no build, test, or CI — the only "command" is the installer.

## Common commands

- Install (or sync) everything: `./install.sh` — runnable from any cwd; idempotent, safe to re-run. Reports Brewfile drift (packages installed but no longer listed) without removing anything.
- Apply Brewfile removals: `./install.sh --cleanup` — runs `brew bundle cleanup --force`, uninstalling any brew/cask not in the Brewfile.
- Reload shell after editing `zsh/.zshrc`: `reload` (alias for `source ~/.zshrc`).
- Add a Homebrew package: edit `homebrew/Brewfile`, then `brew bundle --file homebrew/Brewfile`.

## Architecture

`install.sh` is the orchestrator. After a `preflight` (macOS + `brew` checks) it runs: `brew bundle` → `install_oh_my_zsh` → `install_zsh_plugins` → `install_zsh_config` → `install_aliases` → `install_git_config` → `install_git_signing` → `install_ghostty_config`. Plugins land before `.zshrc` so a shell opened mid-install never references missing plugins.

Configs are **symlinked**, not copied — the repo is the source of truth. Editing `zsh/.zshrc` (or any other tracked config) takes effect in new shells immediately, no re-install needed:

- `zsh/.zshrc` → `~/.zshrc`
- `zsh/aliases.sh` → `~/.config/zsh/aliases.sh` (path `.zshrc` already sources)
- `ghostty/config` → `~/.config/ghostty/config`
- `git/config` → `~/.config/git/config` (XDG path; git finds it without `~/.gitconfig`). Generic — no identity. Includes `~/.config/git/identity` and has `includeIf` rules pointing to `~/.config/git/identity.personal` for `~/Developer/personal/` and `~/Developer/dotfiles/`.
- `git/ignore` → `~/.config/git/ignore` (git's default `core.excludesfile`)
- Oh My Zsh plugins (`zsh-autosuggestions`, `zsh-syntax-highlighting`, `zsh-completions`) are `git clone`d into `$ZSH_CUSTOM/plugins/` and `git pull --ff-only` on re-run.

**Untracked, generated** (kept out of the repo because they contain identity or per-machine secrets):

- `~/.config/git/identity` — default identity + signing setup. Stub written by `install_git_config` on first run; user fills in real `name`, `email`, optionally toggles `commit.gpgsign`. Re-running install **never** clobbers an edited file.
- `~/.config/git/identity.personal` — personal email override, loaded for `~/Developer/personal/` and `~/Developer/dotfiles/` via `includeIf` in `git/config`.
- `~/.config/git/allowed_signers` — generated by `install_git_signing` from the current SSH pubkey + the resolved `user.email`. If the email is still a `TODO` placeholder, the step is skipped with a warning so the user knows to edit the identity file and re-run.
- `~/.ssh/id_ed25519{,.pub}` — generated by `ssh-keygen` on first run (interactive passphrase).

The first-run flow is therefore: `./install.sh` → edit `~/.config/git/identity` (and `identity.personal`) with real values → `./install.sh` again to populate `allowed_signers` → upload `~/.ssh/id_ed25519.pub` to GitHub as auth + signing key.

The `link_file` helper handles the symlink lifecycle: if the target already points at the right source it's a silent no-op; if a real file (or wrong-target symlink) is in the way, `backup_file` snapshots it as `<path>.bak.<epoch>` (using `cp -RP` to preserve symlinks) before replacing. Add new installs by calling `link_file "$SCRIPT_DIR/<src>" "<dst>"` from a new `install_*` function — don't reintroduce `cp -f`.

### Non-obvious wiring

- **`.zshrc` expects the Ghostty CLI on PATH at `/Applications/Ghostty.app/Contents/MacOS`** and adds it explicitly. The Brewfile installs Ghostty via `cask`, so this path assumes the cask install location.
- **Plugin order in `.zshrc` matters.** `zsh-syntax-highlighting` must be loaded last among the zsh-users plugins (Oh My Zsh convention); preserve the existing ordering when adding plugins.
- **Two TERM/COLORTERM exports exist** in `.zshrc` (Ghostty integration block and the env-vars block). They are redundant but harmless — don't "fix" by removing one without checking both blocks.
- **Local overrides:** `.zshrc` sources `~/.zshrc.local` and `~/.zshrc.work` at the end if present. Machine-specific tweaks belong there, not in the tracked `.zshrc`.

### Conventions

- Shell scripts use `set -euo pipefail` and the colored `print_status` / `print_success` / `print_warning` / `print_error` helpers in `install.sh`. Reuse them rather than raw `echo` when extending the installer.
- The Brewfile is grouped by comment headers (`# programming languages`, `# tooling`, `# apps`, `# fonts`). Keep new entries in the right section.
